Напоминание


Некоторым скриптам, логичнее находиться рядом с группой элементов, для которых написан этот скрипт.

Например, скрипт проверки валидации формы, логичнее расположить рядом с формой и элементами формы.

Такие скрипты находятся в HTML-разделе с суффиксом - JS

Ползунок v1 на JS

Обязательно проверять кросс-браузерность.
В списке подозреваемых: IE и EDGE.

Все делает скрипт scroll-add-class.js.

Составные части:

  • HTML - добавить в проект разметку - скопировать из библиотеки;
  • CSS - добавить в проект CSS-файл range.scss;
  • JS - добавить в проект JS-файл custom-range.js;
  • Все.

Принцип работы:

JS-файл блока:

  • для .range__input добавляет opacity = '0';
  • в HTML-файл, после .range__input добавляет разметку с классами:
    • .range__wrapper и .range__wrap-inner- обертки;
    • .range__track - фоновая линия ползунка;
    • .range__progress - линия отображения value;
    • .range__handle - перемещаемый элемент ползунка, т.е. thumb;
    • .range__value -используется для вывода значения value. Если не нужен, удалить из скрипта;
  • JS-файл custom-range1.js отлавливает значение value у штатного input,
    изменяет ширину .range__progress и координату по оси X для .range__handle;
  • Все.

P.S.Ширина .range__progress изменяется от правого края. Это нужно для того, чтобы .range__track можно было дать заливку градиентом, и при изменении value была видна только часть градиента.

Бургер

  1. На странице может быть один или несколько бургеров.
  2. Бургер может просто анимироваться или также открывать другой блок по клику на бургер.

1. Бургер одинокий - один на странице.

Анимированное превращение в крестик по добавлению модификатора:

JS блока burger.js по клику на бургере, добавляет модификатор бургеру.

По клику на бургер, расположенный ниже, представлен код такого простейшего случая.

            var burger = document.querySelector(".burger");
             
            /*события по клику на burger*/
            burger.addEventListener("click", function (event) {
              event.preventDefault();
             
            /* добавляем модификатор на burger, бургер анимируется в крестик */
              burger.classList.toggle("burger--close");
            });

2. Бургер одинокий - открывает/закрывает другой блок, н-р. меню или блок с кодом.

  1. Анимированное превращение в крестик по добавлению модификатора:

    JS блока burger.js по клику на бургере, добавляет модификатор бургеру.

  2. Открытие другого блока по клику на бургере:

    JS блока burger.js по клику на бургере, переключает класс на нужном элементе.
    По клику на бургер, расположенный ниже, представлен код такого простейшего случая.

            var burger = document.querySelector(".burger");
            var element = document.querySelector(".ИмяКласса");
             
            /*события по клику на burger*/
            burger.addEventListener("click", function (event) {
              event.preventDefault();
             
            /* добавляем модификатор на burger, бургер анимируется в крестик */
              burger.classList.toggle("burger--close");
            /* добавляем модификатор на element, блок ИмяКласса откр/закр */
             element.classList.toggle("ИмяКласса--shown");
            });

Главная навигация (меню)

На малых ширинах преобразуется в мобильный вид, скрывается/показывается по клику на бургер, если JS включен.

Состояние без включенного JS:

  • моб.меню - открыто,
  • бургер скрыт - в разметке есть .burger--no-js.
  • На десктопной ширине бургер - скрыт,
    меню принимает горизонтальный вид - .main-nav.scss.

Состояние при работе JS:

  • моб.меню - скрыто, скрипт добавляет .main-nav__list--js.
  • бургер - показан, - скрипт удаляет .burger--no-js.
  • по клику на бургер - скрипт изменяет высоту меню и модифицирует бургер.
  • На десктопной ширине бургер - скрыт,
    меню принимает горизонтальный вид - .main-nav.scss.

Блоки в разметке:

  • .main-nav
    • html: разметка готова в этом шаблоне "main-nav".

  • .logo
    • html: использовать нужную разметку из шаблона "logo", в зависимости от макета.
    • css: подключить logo.scss.
    • js: нет.

    • html: разметка готова в этом шаблоне "main-nav".

Ссылка "вверх", JS

Ссылка «вверх» в нижнем правом углу страницы.

Cкрипт to-top.js.

Принцип:
  • Делаем классическую разметку для ссылки с href="#некий_id" + плюс расставляем в нужном месте (вверх страницы) этот id.
  • Если JS не включен у пользователя, тогда ссылка видна сразу.
  • Если JS включен, добавляем модификатор "*--no--visible" для ссылки (ссылка не видна).
  • Далее, отлавливаем событие по скролу.
  • Если скролл больше указанного в Javascript-файле блока значения, то удаляем модификатор "*--no--visible" у ссылки.
  • Если скролл меньше указанного в Javascript-файле блока значения, добавляем этот же класс элементу (т.е. ссылке).
  • Все.
P.S.

Можно просто сделать ссылку на id + использовать CSS - свойство scroll-behavior: smooth;.
Но!
"Это не работает в IE + EDGE.
Т.е. в IE + EDGE не будет плавной прокрутки к якорю.

Слайдер

Адаптивность делать медиаусловиями.

Состоит из двух ключевых сущностей:
.slider - "скелет" слайдера. Содержит:

  1. обертку-контейнер всех кадров слайдера .slider__list;
    Эта обертка нужна, например, для анимации кадров слайдера перемещением влево-вправо (относительно "окна", которым в таком случае будет являтся .slider + overflow: hidden; ).
  2. отдельный кадр слайдера .slider__list-item;
  3. по умолчанию кадры скрыты .slider__list-item {display: none;}, показ кадров при добавлении модификатора .slider__list-item--active {display: block;} ;
  4. две кнопки: "назад" - .slider__prev, и "вперед" - .slider__next;
    - позиционируются относительно контейнера .slider
  5. обертка-контейнер кнопок-индикаторов "1-2-3-..." .slider__toggles;
    - позиционируются относительно контейнера .slider
  6. сами кнопки-индикаторы "1-2-3-..." .slider__toggle.

.block-content - контентный блок страницы, в котором находится слайдер.
Стили этого блока задают расположение блока со слайдером, внешний вид и расположение кнопок слайдера.

Переключение слайдов, при необходимости, делать на JS.

Google Maps

Как добавить Google Maps на страницу.

  • если просто карту с дефолтным маркером - использовать iframe
     Быстрый старт: Документация Google width iframe
  • если карту с кастом-маркером, маштабом и т.д. API-key + JS
     Быстрый старт: Документация Google width API-KEY

    Внимание!
    При добавлении Google Maps на страницу, помимо API-key, нужно привязать платежную карту.
    Поэтому, API-key у владельца сайта должен быть свой.

Стилизация: https://mapstyle.withgoogle.com/
