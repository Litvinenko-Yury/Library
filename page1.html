Подключение шрифтов

Как подключить несколько файлов одного шрифта и использовать одно название шрифта.

Нередко, веб-шрифт — это несколько файлов: один файл — одно начертание:

  • файл_1 — плотность 400, нормальный;
  • файл_2 — плотность 400, италик, и так далее.

А хочется везде при указании шрифта использовать одно его название, без упоминания в названии слов normal, italic, light и т.п., а указывать это в соответствующих CSS-свойствах.

Подключаем кучу файлов шрифта с указанием одного и того же имени, но разных font-weight и font-style:

Пример-1 - теория: 
                
@font-face {
  font-family: 'Web font';           /* название одно */
  src: url('**webfont8321.woff2**'); /* разные файлы */
  font-weight: **400**;              /* разная плотность */
  font-style: normal;
 }

 @font-face {
  font-family: 'Web font';           /* название одно */
  src: url('**webfont___1.woff2**'); /* разные файлы */
  font-weight: **700**;              /* разная плотность */
  font-style: normal;
}

Псевдоклассы состояний


Полный порядок псевдоклассов такой: :link, :visited, :hover, :active и :focus.


Удобно:
миксин Sass(Less), например, для ссылок:

              
@mixin links {
    & {
        color: $цвет1;
    &:visited {
          color: $цвет2;
        }
    &:hover {
        color: $цвет3;
        }
    &:active, &:focus {
        color: $цвет4;
    }
  }
}

Использование:

              
a {
    @include links;
}


:link

Псевдокласс :link представляет нормальное состояние ссылок и используется для выделения ссылок, которые до сих пор не посещены. В стилях рекомендуется объявлять этот псевдокласс перед всеми остальными классами этой категории. Этот псевдокласс можно пропустить.


:visited

Используется для стилизации ссылок, уже посещенных пользователем.


:hover

используется для стилизации элемента, на который в данный момент наведен курсор. Это может быть любой элемент, хотя применение этого псевдокласса для ссылок является самым распространенным случаем.


:active

Используется для стилизации “активированных” элементов, путем нажатия мыши или касания к экрану. Активация также может производиться и с клавиатуры, также как в случае с псевдоклассом :focus.


:focus

Используется для стилизации элемента, получившего фокус с помощью курсора, касания или нажатия на клавиатуре. Он часто используется в элементах форм.

Как задавать margin и кастомные стили для БЭМ-блоков?

Основная идея:

  • есть "базовая (голая)" кнопка;
  • в зависимости от родительского блока (например, кнопка в хедере), задаем oтступы и кастомные стили.

Мы может сделать так:

              /*вложенность (но не больше 1 уровня)*/
              .header .btn {...}

            .header__btn {...}
              .btn {...} /*кнопка, которая может использоваться где угодно на сайте*/

Когда создавать модификатор, когда — микс?

источник - БЭМ

Внешний вид блока можно изменить при помощи модификаторов или миксов.

Используйте модификаторы
Если существует вероятность переиспользовать блок в данном оформлении.

Используйте миксы
Если блок имеет специфичное оформление только для данного окружения и не будет переиспользован на проекте.

Grid для IE

IE11 поддерживает старую спецификацию grid.

Поэтому:

  • Нужны префиксы.
  • Нужно отдельно прописывать стили, начиная с самого определения display:gird; и ниже: display: -ms-grid;.
  • Дублировать все свойства для IE. При этом они могут называться иначе!
  • Вместо высоты лучше указывать высоту рядов.
  • IE требует явно прописывать ячейки грида.
  • Важно! IE кладёт все элементы grid-контейнера в самую первую ячейку, то есть значения -ms-grid-column: 1 и -ms-grid-row: 1 для каждой ячейки в IE стоят по умолчанию.
  • Для объединения ячеек надо использовать -span в соответствующем виде.
  • Важно! grid + picture + какой-либо полифил, типа object-fit-polyfill или objectFitImages, в IE не работает корректно.
    Это связано видимо, с рассчетом размеров контейнера, которые IE не может корректно рассчитать. Из-за этого изображения либо нет - полифил object-fit-polyfill) , либо нарушается grid-сетка - полифил objectFitImages.
    Решение: если нужна grid-сетка для для IE + изображения в ячейках, тогда в ячейках не использовать picture, а ложить изображения фоном.

Стилизация скроллбара...

Правильное решение - стилизовать с помощью CSS.

Менее правильное решение - если нужно более кросс-браузерное решение, использовать плагины.

Терминология

Скроллбар состоит из семи различных элементов:

  • ::-webkit-scrollbar { /* 1 - скроллбар */ }
  • ::-webkit-scrollbar-button { /* 2 - кнопка */ }
  • ::-webkit-scrollbar-track { /* 3 - трек */ }
  • ::-webkit-scrollbar-track-piece { /* 4 - видимая часть трека */ }
  • ::-webkit-scrollbar-thumb { /* 5 - ползунок */ }
  • ::-webkit-scrollbar-corner { /* 6 - уголок */ }
  • ::-webkit-scrollbar-resizer { /* 7 - изменение размеров окна */ }

Слайдер

Адаптивность делать медиаусловиями.

Состоит из двух ключевых сущностей:
.slider - "скелет" слайдера. Содержит:

  1. обертку-контейнер всех кадров слайдера .slider__list;
    Эта обертка нужна, например, для анимации кадров слайдера перемещением влево-вправо (относительно "окна", которым в таком случае будет являтся .slider + overflow: hidden; ).
  2. отдельный кадр слайдера .slider__list-item;
  3. по умолчанию кадры скрыты .slider__list-item {display: none;}, показ кадров при добавлении модификатора .slider__list-item--active {display: block;} ;
  4. две кнопки: "назад" - .slider__prev, и "вперед" - .slider__next;
    - позиционируются относительно контейнера .slider
  5. обертка-контейнер кнопок-индикаторов "1-2-3-..." .slider__toggles;
    - позиционируются относительно контейнера .slider
  6. сами кнопки-индикаторы "1-2-3-..." .slider__toggle.

.block-content - контентный блок страницы, в котором находится слайдер.
Стили этого блока задают расположение блока со слайдером, внешний вид и расположение кнопок слайдера.

Переключение слайдов, при необходимости, делать на JS.
